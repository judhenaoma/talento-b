Seleccin del modelo de recomendacin

De acuerdo al requerimiento dado en la prueba tcnica, se debe implementar un modelo de recomendacin que permita sugerir libros similares a un libro dado, de acuerdo a esto, entendiendo de que el concepto de similaridad es dado por el contenido del libro. Para ello, se ha decidido implementar un modelo de recomendacin basado en contenido.

El dataset dado para la prueba tcnica contiene informacin propia del libro, tal como descripcin, autores, ttulo, categora, editorial, entre otros. Por otra parte, contiene reseas que los usuarios han dejado sobre el libro y una puntuacin. A la hora de seleccionar las variables he decidido utilizar las variables que objetivamente describen el contenido del libro, tales como la descripcin, autores, ttulo, categora y editorial. Las variables de reseas, rating y otras no sern utilizadas en el modelo de recomendacin, ya que no aportan informacin objetiva sobre el libro y dan cuenta de descripcin propias de los usuarios. Si bien es cierto que en algunos casos las reseas pueden ser tiles para el modelo de recomendacin, en este caso no se utilizarn.

Por ejemplo, he encontrado reseas como:

  • Really Enjoyed It
  • How could someone not love this ??
  • This was a great book

Las cuales no sera tiles para recomendar un libro similar a otro, pero quiz s para recomendar libros que le puedan gustar a un usuario en particular de acuerdo a sus preferencias previas.

Seleccin de variables

He decidido seleccionar las siguientes variables:

  • Ttulo: El ttulo del libro. Hay libros que tienen ttulos muy descriptivos y que pueden dar una idea de lo que trata el libro, adems, hay otros libros que son del mismo autor y que tienen ttulos similares.
  • Descripcin: La descripcin del libro. Este contiene palabras clave que describen el contenido del libro y es un resumen de lo que trata el libro.
  • Autores: Los autores del libro. Hay autores que tienen un estilo particular y que tratan temas similares en todos los libros que escriben.
  • Categora: La categora a la que pertenece el libro. Es ms probable que libros de la misma categora sean similares entre s.
  • Editorial: La editorial del libro. Hay editoriales que se especializan en un tipo de libros en particular y que tienen un estilo particular.

TfidfVectorizer

Se busca convertir el texto en vectores numricos para poder ser procesados, por lo tanto se obtiene una matriz que tiene un valor de 0 a 1 la importancia de cada termino con relacin al vocabulario de todos los datos o corpus.

Las stopwords se eliminan del texto, las cuales son palabras sin relevancia semntica como 'of', 'the', 'in' etc. son eliminadas por el algoritmo.

Por otra parte, la mtrica de similitud que se utiliza es la similitud coseno, la cual mide la similitud entre dos vectores de la matriz conformada por el algoritmo TfidfVectorizer.

def recomendar_libros(titulo_libro, muestra_libros, cosine_sim):
    _muestra_libros = muestra_libros.copy()

    # Crear una serie con los ndices de los libros y los ttulos
    indices = pd.Series(_muestra_libros.index, index=_muestra_libros['Title']).drop_duplicates()
    
    # Obtener el ndice del libro dado
    if titulo_libro not in indices:
        raise ValueError(f"El libro '{titulo_libro}' no se encuentra en la muestra.")
    indice_libro = indices[titulo_libro]
    
    # Crear un DataFrame con los puntajes de similitud
    puntajes_similitud = pd.DataFrame(cosine_sim[indice_libro], columns=["score"])
    
    # Obtener los ndices de los libros ms similares, excluyendo el propio libro
    puntajes_similitud = puntajes_similitud.drop(indice_libro)
    indice_libro = puntajes_similitud.sort_values("score", ascending=False).head(3).index
    
    # Crear el DataFrame de libros recomendados con los puntajes de similitud
    libros_recomendados = _muestra_libros.iloc[indice_libro].copy()
    libros_recomendados['score'] = puntajes_similitud.loc[indice_libro].values
    
    return libros_recomendados[['Title', 'score']]

Anlisis Latente Semntico (LSA)

El Anlisis Latente Semntico (LSA) es una tcnica de reduccin de dimensionalidad que se utiliza para encontrar relaciones semnticas entre palabras y textos. Para emplearse se recurre a la descomposicin en valores singulares (SVD) a partir de una matriz TF-IDF vista anteriormente

In[]:
# Convertir las palabras a una matriz de token de palabras de 0 y 1
vectorizador = CountVectorizer()
sopa_palabras = vectorizador.fit_transform(muestra_libros['cuerpo_modelo'])
# Convertir la matriz de token de palabras a una matriz de frecuencia de palabras
modelo_tfidf_trans = TfidfTransformer()
modelo_tfidf = modelo_tfidf_trans.fit_transform(sopa_palabras)

# Reduccion de dimensionalidad con LSA
lsa = TruncatedSVD(n_components=100, algorithm='arpack')
lsa.fit(modelo_tfidf)

# Se obtiene la matriz de similaridad entre los libros
cosine_lsa = cosine_similarity(modelo_tfidf, modelo_tfidf)

Redes neuronales

Con el objetivo de mejorar la calidad de las recomendaciones se decide implementar un modelo de recomendacin basado en redes neuronales, con el fin de encontrar relaciones entre los libros de manera semntica, ms all de una relacin de frecuencia dada por los modelos anteriores. Se utiliza un modelo utilizado en Procesamiento de Lenguaje Word2Vec para encontrar las representaciones latentes de los libros y poder recomendar libros similares.

model = Word2Vec(vector_size=100, window=5, min_count=1, workers=4)
# Construccin del vocabulario
model.build_vocab(muestra_libros['cuerpo_modelo'])

# Funcin para promediar los vectores de palabras para un texto
def promediar_vectores_palabras(palabras, modelo, vocabulario, num_features):
    vector_caracteristicas = np.zeros((num_features,), dtype="float64")
    n_palabras = 0.
    for palabra in palabras:
        if palabra in vocabulario: 
            n_palabras += 1.
            vector_caracteristicas = np.add(vector_caracteristicas, modelo.wv[palabra])
    
    if n_palabras:
        vector_caracteristicas = np.divide(vector_caracteristicas, n_palabras)
    return vector_caracteristicas

# Funcin para calcular los vectores de palabras promedios para todos los textos en un corpus
def vectorizador_promedio_palabras(corpus, modelo, num_features):
    vocabulario = set(modelo.wv.index_to_key)
    caracteristicas = [promediar_vectores_palabras(oracion_tokenizada, modelo, vocabulario, num_features) for oracion_tokenizada in corpus]
    return np.array(caracteristicas)


# Se calcula el vector promedio para cada libro
w2v_arr = vectorizador_promedio_palabras(corpus=muestra_libros['cuerpo_modelo'], modelo=model, num_features=100)

def recomendar_libros_rn(titulo_libro, muestra_libros, w2v_arr):
    _muestra_libros = muestra_libros.copy()

    # Crear una serie con los ndices de los libros y los ttulos
    indices = pd.Series(_muestra_libros.index, index=_muestra_libros['Title']).drop_duplicates()
    
    # Obtener el ndice del libro dado
    if titulo_libro not in indices:
        raise ValueError(f"El libro '{titulo_libro}' no se encuentra en la muestra.")
    indice_libro = indices[titulo_libro]
    
    vector_libro = w2v_arr[indice_libro].reshape(1, -1)
    cosine = cosine_similarity(vector_libro, w2v_arr)
    
    # Crear un DataFrame con los puntajes de similitud
    puntajes_similitud = pd.DataFrame(cosine[0], columns=["score"])
    
    # Obtener los ndices de los libros ms similares, excluyendo el propio libro
    puntajes_similitud = puntajes_similitud.drop(indice_libro)
    indice_libro = puntajes_similitud.sort_values("score", ascending=False).head(3).index
    
    # Crear el DataFrame de libros recomendados con los puntajes de similitud
    libros_recomendados = _muestra_libros.iloc[indice_libro].copy()
    libros_recomendados['score'] = puntajes_similitud.loc[indice_libro].values
    
    return libros_recomendados[['Title', 'score']]

Dificultades

  • La cantidad de datos es muy grande y el entrenamiento de los modelos es muy costoso computacionalmente, por lo tanto se decidi utilizar el subconjunto de datos que fue posible para entrenar los modelos.
  • No fue posible realizar validacin de los modelos debido a que no se tiene un conjunto de datos de prueba y no fue posible obtener conjuntos de datos de recomendacin de modelos maduros, por lo tanto no se pudo evaluar la calidad de las recomendaciones. Se emple intuicin y el conocimiento del dominio para tener una ida calidad de las recomendaciones, las cuales fueron consistentes en los 2 de los 3 modelos implementados.
  • Aunado a lo anterior, dado que se tiene un subconjunto de datos para entrenar los modelos, es posible que los modelos no sean lo suficientemente robustos para hacer recomendaciones de calidad, ya que muchos libros no estn presentes en el subconjunto de datos.

Recomendaciones

  • Tener disponibilidad de un conjunto de datos de prueba para evaluar la calidad de las recomendaciones y as poder entregar mtricas optimas de los modelos.
  • En el caso de los sistemas de recomendacin usando redes neuronales, se recomienda usar modelos pre-entrenados para obtener mejores resultados.

Conclusiones

  • Se implementaron 3 modelos de recomendacin basados en contenido para recomendar libros similares a un libro dado. 2 de los modelos implementados son modelos de recomendacin clsicos, TfidfVectorizer y Anlisis Latente Semntico (LSA), con estos se obtuvieron recomendaciones consistentes y de calidad; el otro modelo implementado es un modelo de recomendacin basado en redes neuronales el cual obtuvo resultados poco consistentes.
  • A pesar de contar con un subconjunto de datos para entrenar los modelos, se obtuvieron recomendaciones consistentes y de calidad para los libros dados en la prueba tcnica.
